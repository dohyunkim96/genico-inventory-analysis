# 📦 실무 재고관리 데이터 분석 프로젝트  
### 안전재고 · 소비량 · 위험점수 기반 품목 관리

---

## 📝 프로젝트 개요  
본 프로젝트는 실제 식품 재고관리 데이터를 기반으로  
재고 부족 위험을 정량적으로 판단하고,  
보충 우선순위를 도출하기 위한 분석 프로젝트입니다.  

단순 재고 수량이 아닌 **소비 속도와 패턴을 반영한 위험 기준**을 설계하여  
실무에서 바로 활용 가능한 의사결정 기준을 만드는 것을 목표로 했습니다.

---

## 🚨 문제 정의  

기존 재고관리 방식에는 다음과 같은 문제가 있었습니다.

- 날짜별 시트로 분리되어 있어 전체 흐름 파악이 어려움  
- 현재 재고가 위험한 수준인지 판단 기준이 없음  
- 단순 재고 수량만으로는 실제 부족 위험을 설명하기 어려움  
- 어떤 품목을 먼저 보충해야 하는지 기준이 없음  
- 반복적인 수작업 확인으로 업무 효율이 낮음  

👉 따라서,  
**재고 위험도를 수치화하고 우선순위를 정하는 기준이 필요했습니다.**

## 📐 핵심 로직  

### 🔹 안전재고 정의  
- 전월 총 출고량 기준 2주치 재고를 안전재고로 설정  

---

### 🔹 상태 분류 기준  

- 코업 재고 ≤ 안전재고 / 3 → 🔴 긴급  
- 안전재고 / 3 < 코업 재고 < 안전재고 → 🟡 주의  
- 코업 재고 ≥ 안전재고 → 🟢 정상  

---

### 🔹 위험등급 기준  

- 위험점수 ≥ 200 → 즉시처리  
- 위험점수 ≥ 100 → 우선처리  
- 위험점수 ≥ 50 → 주의  
- 그 외 → 여유  

---

## 📈 주요 인사이트  

- 단순 재고 수량보다 **소비 패턴을 함께 고려해야 실제 위험 품목 식별이 가능**  
- 동일한 재고 수준에서도 최근 출고량 변화에 따라 위험도가 크게 달라짐  
- 상태 분류는 빠른 판단에 유용하고,  
  위험점수는 **우선순위 결정에 더 효과적**  
- 실무에서는 “재고 부족 여부”보다  
  **“어떤 품목을 먼저 대응해야 하는가”가 더 중요함**
